Production Resilience

Agents fail gracefully. Systems recover automatically.

Exponential Backoff on API errors
RateLimitError → 5s / 10s / 20s retry. InternalServerError → 2s / 4s / 8s. ConnectionError → 3s / 6s / 12s. Agents survive Anthropic service blips without losing task context.
Loop Detection Circuit Breaker
If the same tool with identical inputs runs 3× in the last 4 calls, the loop breaks and an alert is sent to EM. Prevents infinite retry spirals that waste tokens and time.
Exhaustion Rescue
When an agent exhausts its 25-iteration loop without naturally completing, an HTTP POST fires to the EM's inbox: "Agent stuck — here's the last context — reassign or continue." No task is ever silently abandoned.
Message Durability (XAUTOCLAIM)
On startup, every agent claims orphaned pending messages from any previous dead consumer. An agent that crashes mid-task and restarts picks up exactly where it left off — no message dropped.
Final-Stretch Warning
At iterations 23, 24, 25 of 25, agents receive a system hint: "You have ~N steps remaining — wrap up, don't start new work." Prevents half-finished commits.
Thread Close Guard
POST /threads/{id}/close returns HTTP 409 if any tasks are still pending or in-progress. The team can't declare done while work is outstanding. Requires ?force=true override.
