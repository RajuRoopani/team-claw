Model tier strategy
Expensive coordination (PO, EM) runs on Opus. Implementation (Sr Dev, UX) on Sonnet. Execution (Jr Dev) on Haiku. Cost is proportional to cognitive load — exactly like a real team.
Real escalation paths
Jr → Sr (max 3 rounds), Sr → EM on blocker, any agent → human via ask_human. Agents have persistent memory that evolves after every task. They get better over time.
25 specialized tools
File ops, git (commit/push/branch/merge), code execution in sandbox, Kanban management, wiki, per-agent memory, token budget checks, HITL escalation.
Human authority preserved
Agents pause and ask when genuinely blocked. Humans can steer any live task mid-flight via the dashboard chat bar or CLI — no restart required.
Engineering Architecture

Built for production, not for demos.

LayerTechnologyWhy
Message busRedis StreamsPersistent, replayable, per-agent inboxes. XAUTOCLAIM recovers orphaned messages on restart — no silent drops.
StatePostgreSQL 16Full ACID. 9 tables — messages, tasks, CI, telemetry, memories.
Code executionSandbox containerNo network. 512 MB RAM cap. Read-only workspace. Agents can't break each other.
Git strategyPer-task isolationEach task gets its own subdirectory + repo. Zero cross-task contamination across parallel work.
Context mgmtThree-tierLast 5 verbatim → 6–20 summarized → older in DB. Infinite task length without blowing context windows.
API layerFastAPI + SSE38 endpoints. Real-time event stream for dashboard + webhooks for external integrations.
Resilience built in

• Circuit-break loop detection (3× same tool)

• Exhaustion rescue → EM inbox when stuck
• Thread close guard — can't close with open tasks
Full observability
• Every tool call timed (p95 latency tracked)
